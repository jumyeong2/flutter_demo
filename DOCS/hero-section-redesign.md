# Hero 섹션 전면 수정 - 메시지 집중형 구조

## 변경 일자
2025년 1월

## 목표
랜딩페이지의 첫 번째 Hero 섹션을 메시지 집중형 구조로 전면 수정하여, 사용자가 첫 화면에서 하나의 문제 인식만 전달받고 즉시 "이건 나한테 중요한 문제다"라고 느끼도록 설계.

## 핵심 원칙
- **설득이 아닌 집중**: Hero는 '설득'이 아니라 '집중'을 만드는 영역
- **단일 메시지**: 첫 화면에서 하나의 문제 인식만 전달
- **즉각적 공감**: 사용자가 즉시 문제의 중요성을 인식해야 함
- **여백 확보**: 시선 집중을 위한 충분한 여백

## 새로운 구조

### 1. 상단 배지 (Badge)
- **텍스트**: "팀 필수 안전장치"
- **위치**: Hero 상단에 작게 배치
- **스타일**: 시각적 강조 최소화 (보조 요소)
- **상태**: ✅ 유지 (기존 구현 그대로 사용)

### 2. 메인 카피 (Headline)
- **텍스트**: 
  ```
  공동창업은 아이디어보다
  기준이 어긋날 때 무너집니다.
  ```
- **위치**: Hero에서 가장 큰 타이포그래피
- **역할**: 화면의 시각적 중심
- **줄 수**: 2줄 허용
- **변경사항**: 기존 "공동창업은 아이디어보다\n합의에서 먼저 흔들립니다." → 수정

### 3. 서브 카피 (Sub-headline)
- **텍스트**:
  ```
  CoSync는 의견이 아닌
  '기준의 차이'를 먼저 드러냅니다.
  ```
- **위치**: 메인 카피 바로 아래 배치
- **역할**: 설명은 최소화하고 방향성만 제시
- **줄 수**: 2줄 허용
- **변경사항**: ✅ 신규 추가

### 4. CTA (Primary CTA)
- **버튼 텍스트**: "지금 바로 진단 시작"
- **위치**: Hero 내 유일한 행동 버튼
- **변경사항**: 기존 2개 버튼 → 1개로 통합
- **제거**: 보조 CTA (미리보기, 설명 보기 등) 모두 제거

## 제거할 요소

### 완전 제거
1. ✅ **통계 수치 카드** (`_buildStatisticCard`)
   - 43% 통계 인포그래픽
   - 아이콘 및 설명 텍스트
   
2. ✅ **Rulebook 설명 섹션**
   - "공동창업 합의를 구조화하는 룰북 솔루션" 문구
   - "CoSync Rulebook" 브랜딩
   
3. ✅ **기능 리스트/키워드**
   - `_buildKeywordItem` 위젯
   - 체크 아이콘과 키워드 (지분율, 기여도, 권한, 엑시트)
   
4. ✅ **보조 CTA 버튼**
   - "결과 화면 미리보기 (무료)" 버튼
   - 기존 "우리 팀 Rulebook 만들기" 버튼 (새 CTA로 대체)

### 제거할 문구
- "팀이 흔들리기 전에, 지금 기준부터 맞추세요." (CTA 위 문구)

## 디자인 가이드

### 여백 (Spacing)
- 배지와 메인 카피 사이: 충분한 여백 확보
- 메인 카피와 서브 카피 사이: 적절한 간격
- 서브 카피와 CTA 사이: 충분한 여백으로 시선 집중
- CTA 아래: 섹션 종료를 위한 여백

### 타이포그래피
- **메인 카피**: 가장 큰 폰트 사이즈 (기존 Hero 타이포 유지)
- **서브 카피**: 메인 카피보다 작지만 명확히 읽힐 수 있는 크기
- **CTA 버튼**: 명확하고 눈에 띄는 스타일

### 반응형 고려사항
- 모바일/태블릿/데스크톱 모든 화면 크기에서 메시지가 명확히 전달되어야 함
- 기존 반응형 로직 유지 (`isSmallMobile`, `isMobileScreen`, `isSmallScreen`)

## 흐름 연결

### 다음 섹션과의 연결
- Hero 아래 다음 섹션에서 문제의 구조적 심각성(통계, 히든 코스트)을 설명하도록 흐름을 열어둠
- Hero는 문제 인식, 다음 섹션은 문제의 심각성 설명
